## F15 — send-prompt 語音輸入（speech-to-text）

**範圍**：只在 send-prompt textarea；`lang='zh-TW'`；不做 rename/status、不做語言切換。

**UI**：send-prompt 容器（`web/ui/app.js:607` 的 `.send-prompt`）內，textarea 與 `✎ 送出 prompt` 鈕之間加一顆 🎤 鈕。

**機制**：瀏覽器原生 `webkitSpeechRecognition`，`lang='zh-TW'`、`interimResults=true`、`continuous=false`。按下開始辨識（鈕進入錄音態）、再按停止；`onresult` 把辨識文字即時 append 進該卡的 `.send-prompt-input`。**完全沿用既有「不自動 Enter、使用者檢查後手動送出」流程** —— 不碰 `sendPromptToSession`，只是幫忙填字。

**降級**：`!('webkitSpeechRecognition' in window)` → 不渲染 🎤（Firefox/Safari 支援差）。辨識出錯（`onerror`，如未授權麥克風）→ 鈕復位 + 短提示。

**隱私註記（寫入 UI 提示或 SETUP/README）**：Chrome 的 `webkitSpeechRecognition` 會把音訊送 Google 伺服器辨識（需連網）。dashboard 跑 `127.0.0.1` = secure context，麥克風權限 OK。單人自用接受。

**檔案**：`web/ui/app.js`（🎤 鈕 render + recognition 邏輯）、`web/ui/styles.css`（鈕樣式、錄音態）、必要時 `web/dashboard.html`。純前端，不動 server。

**測試**：瀏覽器 API 無法 node 自動測 → **人工清單**（Chrome 點 🎤 → 講話 → 文字進框 → 手動送出；拒絕麥克風 → 鈕復位；非 Chrome → 🎤 不出現）。

## F16 — 自動核准「編譯/測試/安裝」開關（核心、最敏感）

### 本質（先講清楚）
權限詢問是 **Claude Code 自己**在 tool-use 前跑 `PreToolUse` hook 決定的，dashboard 看不到也插不了手。所以「自動核准」**只能由 user-level hook 執行**；dashboard 只是「開關 UI」（touch/rm 旗標檔）。

### 架構
1. **執行端（repo 擁有 + installer 裝到 user-global）**：新增 canonical `hooks/auto-approve-build.sh`，作為**獨立第二支 PreToolUse hook**（**不動**使用者手寫的 `auto-approve.sh`）。
2. **安裝**：最小 `install-hooks.ps1`（仿 `install-autostart.ps1`）——把 `hooks/auto-approve-build.sh` 複製到 `~/.claude/hooks/` 並在 `~/.claude/settings.json` 的 `PreToolUse` 增註一筆。
3. **F16 專屬 capability gate**：`GET /api/auto-approve-build` 回 `{ hookInstalled, state }`；FE 若 `!hookInstalled` → toggle 變灰 + 提示「先跑 install-hooks.ps1」。（v0.3 F23 落地後此 gate 退役改吃通用框架。）

### Hook 放行邏輯（`auto-approve-build.sh`，沿用 `auto-approve.sh` 分層結構）
- **(0) 旗標 gate**：`~/.claude/auto-approve-build.enabled` 不存在 → `exit 0`（defer）。
- **(1) 硬 deny veto（最前、最優先）**：`>`/`>>` 重導、`$()`/反引號 subshell、`&&`/`||`/`;`(非白名單收尾)、`rm/mv/cp/sudo/chmod/kill/...`、install 與其他指令鏈接 → 一律不放行。
- **(2) 放行 pattern**：主指令命中白名單 **且** 只接「無害收尾」：
  - 白名單動詞（低風險）：build（`mvn compile/test-compile/verify`、`gradle build/assemble`、`./gradlew`、`tsc`、`npm|pnpm|yarn run build`、`vite build`、`go build/vet`、`cargo build/check`、`make`、`dotnet build`）；test（`mvn test`、`gradle test`、`npm|pnpm|yarn test`、`jest/vitest/mocha/playwright/cypress`、`go test`、`cargo test`、`pytest`/`python -m pytest`、`dotnet test`）；lint/typecheck（`eslint`、`prettier --check`、`tsc --noEmit`、`ruff check`、`mypy`、`gofmt -l`）。
  - 白名單動詞（⚠️ install，**使用者 2026-06-09 確認納入**）：`npm install/ci`、`pnpm install`、`yarn`、`pip/pip3 install`、`poetry/pipenv install`、`uv pip install`、`go mod download/tidy`、`go get`、`cargo fetch`、`bundle install`、`composer install`、`dotnet restore`。**⚠️ install 會跑 postinstall script = 執行任意程式碼**；故僅在開關 ON 時放行、預設 OFF、寫 audit log。
  - 允許的「無害收尾」：`| tail|head|grep`（純讀取尾管）、`; echo ...`、`; ls ...`。其餘組合一律落回 deny。
  - **網路/抓檔**（`git pull`/`git fetch`/`curl`/`wget`）：**不納**（todo 已定）。
- **(3) audit log**：沿用 `~/.claude/auto-approve.log` 格式。

### 三段式狀態 + 旗標模型（使用者 2026-06-09 定）
- **真旗標** `~/.claude/auto-approve-build.enabled`（hook 讀它）+ **持久標記** `~/.claude/auto-approve-build.persist`。
- 狀態：
  - **關（預設・每次問）**：兩檔皆無。
  - **本次 session 有效**：只有 `.enabled`（dashboard 關服務時清掉）。
  - **永久套用**：`.enabled` + `.persist`（跨重啟；dashboard 啟動時依 `.persist` 重建 `.enabled`，關服務不清）。
- **dashboard 啟動對帳**（`server/index.js` 啟動時）：有 `.persist` → 確保 `.enabled` 在；無 `.persist` → 清掉殘留 `.enabled`（崩潰防呆，回「每次問」）。
- **dashboard 關閉**（`/api/shutdown`、`/api/restart` 前）：無 `.persist` → 清 `.enabled`。
- 隨時可撤銷永久 → 回「關」。

### Server endpoints（`server/index.js`，沿用既有 `/api/*` 風格）
- `GET /api/auto-approve-build` → `{ hookInstalled, state: 'off'|'session'|'permanent' }`。
- `POST /api/auto-approve-build` body `{ state }` → 依 state 設/清兩個旗標檔。FE 不碰 fs。

### FE
- topbar 加 F16 控制：**主開關**（關 ↔ 開＝本次 session）+「**永久套用**」checkbox（勾＝跨重啟；對應 `state=permanent`）。預設關，狀態讀 `GET /api/auto-approve-build`；`!hookInstalled` 時整組變灰 + 提示。

### ⚠️ 實作前必驗證的技術點
「獨立第二支 PreToolUse hook」與現有 `auto-approve.sh` **並存**時，Claude Code 多個 PreToolUse hook 的合成行為（allow/deny 如何疊、是否所有 hook 都跑、precedence）。**寫 plan 階段用 claude-code-guide 查證**。若並存不可靠 → fallback：把 build tier 併入現有 `auto-approve.sh`（犧牲「不動手寫檔」的乾淨度）。

### 檔案
- repo：`hooks/auto-approve-build.sh`（新）、`install-hooks.ps1`（新）、`server/index.js`（endpoints + 啟動對帳 + 關閉清旗標）、`web/ui/app.js`/`styles.css`（toggle）。
- user-global（由 installer 動，非 repo）：`~/.claude/hooks/auto-approve-build.sh`、`~/.claude/settings.json`、旗標檔 `auto-approve-build.{enabled,persist}`。

### 測試
- **hook 測試**（餵 JSON stdin 斷言 stdout）：`mvn test`✅ / `mvn test | tail -5`✅ / `npm install`✅(ON 時) / `mvn test > out`❌ / `npm install && rm -rf x`❌ / `$(curl evil)`❌ / 旗標不存在時全部 defer。
- **server 狀態機**：node:test 驗 off/session/permanent 三態的旗標檔結果 + 啟動對帳。

## F21 — 新 session：資料夾 modal → 開終端跑 claude

**範圍**：彈出 modal、目錄樹逐層進入、選定目標夾、**記憶上次選擇**、開終端跑 `claude`。

### Server（`server/index.js`，新 endpoints）
- `GET /api/fs/list?path=<dir>` → `{ path, parent, drives?, dirs:[名稱…] }`。`path` 省略 → 回 Windows 磁碟機根清單（列 `C:\`、`D:\`…）。用 `fs.readdirSync(dir,{withFileTypes:true})` 濾 `isDirectory()`；隱藏/系統夾可選擇略過。錯誤（無權限/不存在）→ 400 + message。
- `POST /api/launch-session` body `{ dir }` → 驗證 dir 存在且為目錄 → `spawn` 開新終端跑 claude，detached/unref（複用 `server/index.js:324-349` 模式）。指令：優先 `wt -d <dir> cmd /k claude`（Windows Terminal），偵測不到退回 `start` + `powershell -NoExit -Command "Set-Location -LiteralPath '<dir>'; claude"`。

### FE（`web/ui/app.js` + `styles.css` + `dashboard.html`）
- topbar「＋ 新 session」鈕 → 開 modal。
- modal：麵包屑（目前路徑、可點上層）+ 目錄清單（雙擊進入子夾、單選選定）+「在此開啟」鈕 + 取消。
- **記憶上次**：localStorage 存最後選定/瀏覽路徑；下次開 modal 定位到該層（不存在則退磁碟機根）。
- **啟動前確認**：按「在此開啟」後 confirm（「將在 \<dir\> 開新 Claude session」）再打 `/api/launch-session`。

**安全**：任意目錄 spawn = 跑任意程式；但本服務 127.0.0.1 單人自用、使用者主動觸發 + 確認框 → 接受。

### 測試
- `GET /api/fs/list`：node:test（給臨時目錄，斷言列出子夾、`parent` 正確、省略 path 回 drives）。
- launch：人工清單（modal 選夾 → 終端真的開起來跑 claude；wt 有/無兩種環境）。

## 整體驗證（end-to-end）
1. `cd server && node index.js`，瀏覽器開 `http://127.0.0.1:7878/`。
2. F15：Chrome 展開一張卡 → 🎤 → 講話 → 文字進 send-prompt → 手動送出。
3. F21：topbar ＋ → modal 逐層進入選夾 → 確認 → 新終端跑起 claude；重開 modal 記得上次路徑。
4. F16：跑 `install-hooks.ps1` → toggle 由灰轉可用 → 開「本次」→ 在某 session 跑 `mvn test`/`npm install` 不跳權限、`mvn test > x` 仍跳；關 dashboard 後 `.enabled` 被清；設「永久」後重啟 dashboard 仍 ON。
5. `cd server && node --test`（hook 測試 + server 狀態機 + fs/list）。
